## 什麼時候使用

- 製作 PPT、slides、deck、keynote、簡報、分享稿
- 製作可用鍵盤播放的 HTML 簡報
- 製作需要講者模式、逐字稿、動畫或 PNG 匯出的簡報
- 製作小紅書圖文、課程模組、產品發表、技術分享、週報、提案簡報

## 開工前先確認

1. 內容與觀眾：主題、頁數、誰要看、使用情境
2. 風格：正式匯報、技術分享、小紅書、學術報告、產品發表等
3. 起點：選一個 full-deck template 或 single-page layout

## 製作原則

- 從模板開始，不從空白頁硬寫
- 用設計 token 和 theme，不在頁面亂塞固定色碼
- 一頁只放一個清楚重點
- 講者提示放 notes，不放到觀眾看的畫面
- 需要演講或分享時，使用 presenter mode，逐頁寫 150-300 字提示稿
- 最後用瀏覽器實測鍵盤播放、手機比例、大螢幕比例和輸出結果

## 常用模板方向

- 商務提案：`pitch-deck`、`corporate-clean`、`pitch-deck-vc`
- 技術分享：`tech-sharing`、`tokyo-night`、`blueprint`
- 小紅書圖文：`xhs-post`、`xiaohongshu-white`、`soft-pastel`
- 課程：`course-module`
- 週報：`weekly-report`
- 演講逐字稿：`presenter-mode-reveal`

## 交付檢查

- 字夠大，大螢幕和手機都能看
- 圖片沒有變形，人物和產品沒有被裁壞
- 動畫服務內容，不搶戲
- 講者 notes 不外露
- 連結、字體、圖片、音樂授權都可交付
- 需要截圖時，用 render script 匯出 PNG
